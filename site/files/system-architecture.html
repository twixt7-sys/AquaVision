System Architecture

system-architecture v1.0.0 updated 2026-07-17 ◎Target raw JSON ↗

Architectural Principles

Principle
Offline is the normal case, not the exception
Implication

Rural aquaculture sites lose connectivity routinely. Every layer must function with the link down and reconcile when it returns. A system that assumes the cloud is a system that fails on the day it matters.

Citations
ai-aqua-review-2024
Principle
Measurements and predictions live in separate namespaces
Implication

The moment a model output can be mistaken for an observation, the dataset is permanently compromised and every downstream analysis is suspect.

Principle
Raw data is immutable
Implication
Corrections are new records referencing the original. Never overwrite a measurement.
Principle
The edge decides what matters; the cloud decides what it means
Implication

Inference and data reduction on board. Training, twin simulation, and cross-site analytics in the cloud.

Principle
The farmer's data belongs to the farmer
Implication

Constrains the architecture: per-tenant isolation, exportability, and deletion must be real capabilities, not policy statements. See ethics-data-privacy.json.

Layers

Layer
Edge — on aircraft
Responsibilities
  • Flight control and failsafe
  • Sensor acquisition and timestamping
  • QC flagging at source
  • NDWI/NDCI computation
  • Anomaly detection inference
  • Data reduction and compression
  • Local buffering
Why Here

Raw multispectral imagery is far too large to upload over an intermittent cellular link, and a bloom detection that requires backhaul fails exactly when the network is congested or down.

Constraint
Every watt of compute comes directly out of endurance. See power-energy-budget.json.
Layer
Edge — dock
Responsibilities
  • Sortie scheduling
  • Store-and-forward
  • Local dashboard for on-site staff
  • Calibration reference management
  • Fleet health
Why Here

The dock has mains or solar power and no mass constraint. Anything that does not need to be on the aircraft should be here rather than in the cloud — it is closer to the operator and it works when the link does not.

Requirement
7 days of local retention minimum. The pre-dawn profile must survive a link outage.
Layer
Platform — cloud
Responsibilities
  • Ingestion and validation
  • Time-series storage
  • Sensor fusion — probe calibrating imagery
  • Digital twin simulation
  • Model training
  • Cross-site analytics
  • Alerting
  • API
Note

The cloud is where value compounds across sites and time. It is not on the critical path for any single sortie, and it must not become so.

Layer
Presentation
Responsibilities
  • PondTwin visualisation
  • Dashboard
  • Mobile alerts
  • Reports and audit exports

The Fusion Core

Why This Is the Architecture and Not A Feature

The probe-calibrates-the-camera insight from solution-overview.json is not a processing step bolted onto a pipeline. It is the reason the system exists, and the architecture should be shaped around it.

The Flow
  • Imagery arrives as reflectance (already calibrated at the edge against the dock panel and the irradiance log).
  • NDWI masks non-water pixels. Nothing downstream sees a dike or a boat.
  • NDCI computed per water pixel — a relative field, not a concentration.
  • Probe profiles from the same sortie arrive with in-situ chlorophyll at known positions.
  • A local regression fits index to concentration for THIS site, THIS day, THIS illumination.
  • The regression is applied across the masked field, producing a concentration map WITH a propagated uncertainty field.
  • Both the map and its uncertainty enter the twin.
The Critical Design Decision

The regression is site-and-day-local, never global. A single global index-to-concentration model across sites and conditions is the mistake that makes UAV water quality retrievals fail in production. Turbidity, CDOM, and illumination differ enough between two lakes — and between two days on one lake — that a global fit is confidently wrong.

Citations
mishra-ndci-2012 mdpi-drones-inland-review-2024 sciencedirect-uav-multispectral-ponds-2024
The Failure Mode To Guard Against

With few probe stations per sortie, the regression is fitted on three or four points. That is a fragile fit and its confidence interval will often be embarrassingly wide. The honest response is to PROPAGATE that uncertainty into the map and show it, not to hide it behind a smooth colour ramp. A map that looks precise and is not will be believed, and someone will make a decision on it.

Data Flow

Sortie Lifecycle
StepEvent
1Dock schedules sortie against the diel cycle and the glint window
2Aircraft acquires — timestamped UTC, QC-flagged at source
3Edge computes indices and runs inference
4Aircraft docks; data transfers to dock over local link
5Dock forwards to cloud when connectivity permits; buffers when it does not
6Cloud validates against the schema, runs the QARTOD-style QC battery, rejects malformed records rather than storing them
7Fusion produces calibrated concentration fields with uncertainty
8Twin state updates; derived indices recomputed
9Alert rules evaluate; alerts dispatch with evidence attached
10Dashboard and twin reflect the new state
The Latency Requirement

From dock to alert, under 5 minutes when connected. A dawn-profile alert that arrives at 09:00 has missed the window it exists to open.

Storage Model

Time Series
Profiles, telemetry, and derived indices. High write volume, append-only.
Spatial
Imagery mosaics, concentration fields, uncertainty fields. Large objects, low query rate.
Relational
Sites, units, stations, sondes, sorties, users, tenants, authorisations.
Immutability

Raw measurements are append-only. A correction is a new record with a reference to the original and a reason. This is not a purity exercise — it is what makes the dataset defensible in an audit, a dispute, or a thesis.

Retention

Raw data indefinitely. It is the asset. The moat is not the drone; it is three years of profiles on a lake nobody else has measured.

The Offline Design

Why It Is A First Class Concern
Most agritech architectures assume connectivity and degrade badly without it. Ours cannot.
Rules
  • The dock schedules and flies with no cloud contact. Indefinitely.
  • The local dashboard works with no cloud contact.
  • Alerts based on threshold rules fire locally. Only model-based alerts requiring cloud inference are deferred — which is a good reason to keep the safety-critical alerts rule-based rather than model-based.
  • Sync is idempotent and resumable. A dock that reconnects after four days uploads four days of data without duplication or loss.
  • Clock discipline matters: a dock that has been offline still needs UTC accuracy, or the diel analysis is corrupted on reconnection.

Multi Tenancy and Data Boundaries

Requirement
Farm A must never see Farm B's data. Not by convention — by construction.
Why It Is Architectural

Use case UC-10 (lake-scale carrying capacity) requires cross-farm aggregation, and the operator who benefits from the lake-wide analysis is the one who least wants their own stocking visible in it. That tension is not resolvable by a policy document; it must be resolved in the data model.

Approach

Strict per-tenant isolation by default. Cross-farm analytics operate only on explicitly consented, aggregated, and — where possible — differentially private outputs. The default is no sharing.

The Honest Gap

We have not designed this properly. It is flagged here because it is the kind of thing that is cheap now and impossible later. See ethics-data-privacy.json.

Status
·UNRESOLVED — Architectural Decision Required Before the First Multi Farm Deployment

Security

Considerations
  • Command-and-control link integrity. An aircraft that accepts a spoofed command is a weapon. Authenticated, encrypted C2 is not optional.
  • The dock is a physically accessible computer on a lakeshore. Assume it will be opened.
  • Farm production data is commercially sensitive. A breach that reveals a competitor's stocking density is a real commercial harm to a real customer.
  • Imagery contains people, houses, and boats. It is personal data whether we intended to collect it or not.
  • The dispensing interlock must not be reachable from the network. Ever. An attacker who can dispense is an attacker who can poison a pond.
Status
·Not Designed. Named Here So It Is Not Forgotten. A Security Architecture Retrofitted After the First Deployment Is A Security Architecture That Does Not Exist.

Technology Choices

Status
·DELIBERATELY UNSPECIFIED
Reasoning

Naming a database and a cloud vendor at TRL 3 is premature and it is the kind of decision that looks like progress while being irrelevant. The architecture above constrains what is needed: append-only time series at moderate volume, large object storage, offline-tolerant sync, and edge inference at a few watts. Many stacks satisfy those. Choose when there is a real workload to measure, not before.

The Only Real Constraint

It must run in the Philippines with Philippine latency and Philippine data residency expectations, and it must not assume the site has internet.

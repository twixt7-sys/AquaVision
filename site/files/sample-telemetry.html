Sample Telemetry — ILLUSTRATIVE SYNTHETIC DATA

sample-telemetry v1.0.0 updated 2026-07-17 ⚠SAMPLE DATA — NOT REAL raw JSON ↗
ℹDANGER READ THIS FIRST
Banner:

EVERY NUMBER IN THIS FILE IS FABRICATED. IT DESCRIBES NO REAL POND, NO REAL FISH, AND NO REAL SORTIE.

Purpose:

This data exists for ONE reason: so the webpage developer has correctly-shaped records to build and test the dashboard and twin against, before real data exists. It shows the SHAPE of the data, not any fact about the world.

The Absolute Rule:

This data must NEVER appear in a thesis results chapter, a pitch as evidence, or any context implying it is real. The webpage MUST render the illustrative_synthetic badge (see brand-identity.json) prominently on anything derived from it. If this data is ever mistaken for real, the integrity of the entire repository is compromised.

How It Was Made:

Hand-constructed to be physically plausible — a realistic diel DO curve, a realistic thermocline, a realistic post-front DO crash — so the charts look believable while being entirely invented. That plausibility is exactly what makes it dangerous if mislabelled.

The Teaching Point

The thermocline sits at ~3-4 m (temp drops from 28.1 to 25.3). Below it: anoxic (0.2-0.9 mg/L), negative ORP (sulfide-generating), acidifying. The surface looks survivable at 3.1 mg/L. A cold rain that mixes this column delivers the anoxic, sulfide-laden bottom water to the fish. This is the mechanism from the Laguna de Bay literature, illustrated with invented numbers.

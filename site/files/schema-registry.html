AquaVision Knowledge Repository — Schema Registry

schema-registry v1.1.0 updated 2026-07-18 raw JSON ↗

Defines the shape of every recurring object in this repository so that a consuming webpage can render any file generically. Covers the whole AquaVision platform across its three phases (free app, premium tier, Phase 3 OmniDrone/IoT hardware), not a single-product schema. These are JSON Schema (draft 2020-12) fragments plus prose notes.

Conventions

Identifiers

kebab-case, globally unique within the repository, permanent. Labels may be edited freely; ids may not.

Units

SI unless the field name states otherwise. Currency fields always carry an explicit ISO-4217 `currency` sibling. Never store a bare number for money.

Dates
ISO 8601. Full dates as YYYY-MM-DD. Years as integers. Ranges as {start, end}.
Nulls

Use null for 'known to be absent'. Use the string 'unknown' for 'we have not determined this'. Never use 0 as a stand-in for missing.

Citations

Any factual claim carries `citation_ids: []` referencing entries in data/07_research/references-bibliography.json.

Provenance
Mandatory on every quantitative record. See enum below.

Validation Notes

  • This registry is descriptive, not enforced by a build step. If you add a file, add its shape here.
  • A consuming webpage should treat any record missing `provenance` as untrusted and render it greyed out.
  • The `illustrative_synthetic` badge is a load-bearing safety feature of this repository, not decoration. Do not strip it in the UI.

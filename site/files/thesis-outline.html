Thesis / Dissertation Outline

thesis-outline v1.0.0 updated 2026-07-17 ◎Target raw JSON ↗

Note

A chapter structure for an engineering/technology thesis on the OmniDrone concept, mapped to the repository files that supply each chapter. Written for a DESIGN-AND-FEASIBILITY thesis (which is what TRL 3 supports), with a clear path to a results-bearing thesis once validation data exists.

The Honest Framing

What Kind of Thesis This Can Be Now

A design, specification, and feasibility study with a pre-registered validation methodology. It contributes a problem analysis, a system architecture, an identified research gap, and a testable plan. It does NOT contribute empirical results, because none exist.

What It Becomes After Validation

An empirical thesis once Phase 2-3 data exists — the profile-vs-reference agreement, the calibration-uncertainty curve, and the stratification-precursor analysis are all publishable results.

The Discipline

Write in proposal/design tense now. Do not use results language until there are results. A committee will forgive an honest feasibility study; it will fail a study that claims validation it does not have.

The Tempting Error

Presenting the synthetic sample data as results. The illustrative_synthetic data exists ONLY to build the webpage. It must never appear in a results chapter. See sample-telemetry.json.

The Contribution Statement

What This Thesis Genuinely Contributes
  • A structured problem analysis grounding a documented, recurring, mechanistically-understood failure mode in published evidence.
  • A reframing of the monitoring problem as labour-economics rather than instrumentation.
  • A system architecture whose central novelty is self-generated calibration ground truth, addressing a gap the UAV water-quality literature repeatedly identifies.
  • A feasibility analysis (mass, energy, risk) that honestly reports where the concept does and does not close.
  • A pre-registered validation methodology including an ethical resolution to the control-group problem and explicit falsification conditions.
What It Does Not Claim

Empirical validation of any performance or predictive capability. That is future work, and the thesis says so plainly.

Why This Is A Legitimate Thesis

A design-and-feasibility study with a rigorous problem analysis, an identified gap, a specified system, and a pre-registered method is a complete and defensible scholarly contribution at this stage. It becomes a stronger, results-bearing thesis after the validation season — but it does not need to overclaim to be worth defending now.

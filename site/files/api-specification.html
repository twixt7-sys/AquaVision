API Specification

api-specification v1.0.0 updated 2026-07-17 ◎Target raw JSON ↗

Status

Design intent. No API is implemented. Paths and shapes are indicative and will change on contact with a real client.

Design Principles

  • REST over HTTPS, JSON. Boring on purpose. There is nothing in this domain that justifies a novel protocol.
  • Read-heavy. The dashboard and the webpage consuming this repository are the primary clients.
  • Every response that contains a number carries its provenance and its QC status. An API that returns bare numbers invites clients to render them as facts.
  • Tenant isolation is enforced at the API boundary, not in the client. A client bug must not be able to expose Farm B's data to Farm A.
  • Uncertainty is never optional in a response. If a client wants a scalar without an error bar, they can discard the field — but we will not omit it.

Note

Every data response carries this envelope. It is the API-level expression of the repository's provenance discipline.

Meta

The Warnings Field

Clients are expected to render warnings, not swallow them. A warning like 'this concentration field was calibrated on 3 points with an R² of 0.41' is more important than the field itself, and an API that makes it easy to ignore is an API that facilitates overclaiming.

Auth and Tenancy

Model
Per-tenant. A farm's data is visible only to that farm and to explicitly granted parties.
The Hard Case

The lake-scale carrying capacity use case (UC-10) needs cross-farm aggregation. The operator who benefits from the lake-wide picture is the one who least wants their own stocking visible in it.

Position

Cross-tenant reads are not exposed by this API at all. Aggregated lake-scale products are a separate, explicitly-consented pipeline with their own governance. Building a cross-tenant read endpoint 'just for internal use' is how the boundary erodes.

Status
·UNRESOLVED. See Ethics Data Privacy.Json. Flagged Because It Is Cheap To Design Now and Impossible To Retrofit.

What This Api Will Not Do

  • Return a concentration without its uncertainty.
  • Return a model output in the same field as a measurement.
  • Accept a dispensing command without a human authoriser.
  • Expose one tenant's data to another, for any reason, including 'research'.
  • Silently include partial profiles in an aggregate.
  • Provide a person-detection or personnel-tracking endpoint. Not because it is technically hard — because building it would change what this company is.
